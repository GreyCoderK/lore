# Lore Demo Repository

This is an example project showing what a Lore-documented repository looks like.

## What's inside

- `.lorerc` — Minimal Lore configuration
- `.lore/docs/` — 4 real documents generated by Lore

## How to reproduce

```bash
# 1. Install Lore
brew install GreyCoderK/lore/lore

# 2. Create a new project
mkdir my-project && cd my-project
git init

# 3. Initialize Lore
lore init

# 4. Make commits — Lore asks 3 questions after each one
echo "hello" > main.go
git add . && git commit -m "Initial commit"
# → Type? feature | What? Initial commit | Why? Project bootstrap

# 5. Check your corpus
lore list
lore status
```

## Documents in this demo

| Document | Type | Description |
|----------|------|-------------|
| `feature-initial-project-setup.md` | feature | Project bootstrap and scaffolding |
| `feature-add-user-authentication.md` | feature | JWT auth middleware implementation |
| `decision-database-selection.md` | decision | Why PostgreSQL over MongoDB |
| `refactor-extract-auth-middleware.md` | refactor | Auth logic extracted from handlers |

These documents were generated using `lore new` during Lore's own development.
